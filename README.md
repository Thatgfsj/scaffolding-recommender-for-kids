# 🏗️ 脚手架算法 - 儿童短视频推荐系统

> "不是递给孩子他最爱吃的糖果，而是悄悄在他伸手可及的地方放上关于蚂蚁工坊、纸飞机折叠、非洲鼓点的书籍。"

## 📖 项目简介

**脚手架算法**（Scaffolding Recommender）是一个专为儿童设计的短视频推荐系统，旨在对抗当前主流算法带来的**成瘾性设计、信息茧房、价值观侵蚀**等问题。

本系统以**认知发展**和**价值观培育**为核心目标，像一位住在屏幕里的、既博学又幽默的图书馆导师，引导儿童在"最近发展区"内自然拓展兴趣边界。

## 🎯 核心特性

| 特性 | 描述 |
|-----|-----|
| **对抗性目标函数** | 不优化点击率/时长，而是优化认知多样性熵、价值观势能、自然结束率 |
| **强制多样性配额** | 60% 教育相关 + 20% 跨域探索 + 10% 引导链路 + 10% 随机发现 |
| **透明推荐** | 用户可点击"为什么推荐这个？"，查看知识图谱路径 |
| **家长白盒** | 语义输入培育意向，查看周报透明度报告 |
| **安全护栏** | AI泔水内容检测，价值观漂移监控 |

## 🚀 快速开始

### 环境要求

- Python 3.8+
- 无需外部依赖（标准库即可运行）

### 安装

```bash
# 克隆仓库
git clone https://github.com/YOUR_USERNAME/scaffolding-recommender-for-kids.git
cd scaffolding-recommender-for-kids

# 运行简单演示
python demo.py --simple

# 运行完整推演案例
python demo.py
```

### 基本使用

```python
from guided_recommender import create_recommender, AgeGroup

# 创建推荐器（针对6-9岁儿童）
recommender = create_recommender("6-9岁")
recommender.set_user("小明_8岁")

# 获取推荐
recommendations = recommender.get_recommendation(5)
for rec in recommendations:
    print(f"{rec['title']} - 得分: {rec['score']}")
    print(f"  理由: {rec['reason']}")

# 更新用户状态
recommender.update_profile("v001", "natural_end")

# 设置家长培育意向
result = recommender.set_parent_intent("希望孩子多培养好奇心")
print(result['confirmation'])

# 获取家长透明度报告
report = recommender.get_parent_report()
print(report)
```

## 📊 核心指标说明

### 对抗性目标函数 J

```
J = w_A × A - w_B × InfoBubbles(p) + w_C × ΔV + w_D × NaturalEndRate
```

- **A**：短期愉悦度（保底线，确保内容不无聊）
- **B**：信息茧房惩罚（熵损失，过低时触发干预）
- **C**：价值观势能（培育积极价值观的概率提升）
- **D**：自然结束奖励（鼓励主动结束，打破刷屏）

### 动态权重场景

| 场景 | w_A | w_B | w_C | w_D |
|-----|-----|-----|-----|-----|
| 冷启动 | 0.7 | 0.1 | 0.1 | 0.1 |
| 轻度茧房 | 0.4 | 0.3 | 0.2 | 0.1 |
| 重度茧房 | 0.2 | 0.5 | 0.2 | 0.1 |
| 无限刷屏 | 0.2 | 0.2 | 0.2 | 0.4 |

## 🔍 推演案例

### 场景

8岁男孩小明，历史记录全是"玩具开箱"和"搞笑猫狗"，需要引导至观看"小哥白尼液压机械臂DIY"。

### 引导序列

```
步骤1: 玩具开箱 (L1) → 确认兴趣锚点
步骤2: 玩具拆解 (L2) → 引入"拆解"元素
步骤3: 简单机械 (L2-L3) → 展示"轮子里的秘密"
步骤4: 杠杆原理 (L3) → 因果关系递进
步骤5: 液压传动 (L4) → 工程原理升级
步骤6: 液压机械臂 (L4) → 目标达成！
步骤7: 真实应用 (L4-L5) → 巩固拓展
步骤8: 扶光周报 → 反馈闭环
```

### 效果评估

| 指标 | 引导前 | 引导后 |
|-----|-------|-------|
| 认知多样性熵 | 1.2 | 2.5 ↑ |
| 好奇心维度得分 | 0.3 | 0.7 ↑ |
| 平均认知复杂度 | L1 | L4 ↑ |
| 主动结束率 | 35% | 65% ↑ |

## 🤝 设计哲学

> 我们将系统想象为一位住在屏幕里的图书馆导师"扶光"。
>
> 她的目标不是递给孩子他最爱吃的糖果（这会导致蛀牙和偏食），而是悄悄在他伸手可及的地方放上关于蚂蚁工坊、纸飞机折叠、非洲鼓点的书籍。
>
> 孩子是被"吸引"过去的，而非被"塞给"的。她知道什么时候该递上一本更难的书，什么时候该退后一步让孩子自己探索。
