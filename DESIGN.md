## 1. 问题陈述

### 1.1 主流算法的结构性缺陷

当前主流短视频平台以**点击率（CTR）最大化**为核心目标函数，这一策略在成人用户场景下已被验证能有效提升平台收益和用户满意度。然而，当该逻辑直接应用于儿童用户时，产生了系统性的伦理风险：

| 缺陷维度 | 具体表现 | 儿童受影响程度 |
|---------|---------|--------------|
| **成瘾性设计** | 无限滚动、自动播放、即时反馈循环 | 儿童前额叶皮质发育不完整，自控力薄弱，极易形成无意识行为依赖 |
| **信息茧房** | "你喜欢的就推更多" 的闭环强化 | 儿童认知尚在构建期，兴趣边界被算法固化，想象力和好奇心受限 |
| **价值观侵蚀** | 低俗内容/软暴力/不良导向的流量倾斜 | 儿童缺乏批判性思维和价值判断能力，算法偏好与家长期望产生冲突 |
| **时长至上** | 平台收益与用户时长正相关 | 儿童无节制使用影响睡眠、运动、社交等健康发展需求 |
| **算法黑箱** | 拒绝公开推荐逻辑 | 家长无法知情、无法干预、无法修正 |

### 1.2 现有"青少年模式"的局限

当前各平台的青少年模式采用"堵"的策略：限时、限充、内容分级过滤。然而：

- 家长可代为绕过防护
- 未成年用户使用成人账号
- 租号、共用账号等规避行为
- "青少年模式"用户体验差，实际使用率低

**核心矛盾**：现有方案将"网络保护"等同于"限制使用"，忽视了**主动价值培育**的命题。

### 1.3 设计结论

我们需要构建一个**从底层目标函数开始就与主流算法对抗**的新系统：

> 不优化点击率 | 不最大化时长 | 不迎合既有兴趣 | 不闭门造车

而是以**认知发展**和**价值观培育**为核心目标，引导儿童在"最近发展区"（Zone of Proximal Development）内自然拓展兴趣边界。

## 2. 核心隐喻与设计哲学

### 2.1 系统人格设定

我们将脚手架算法系统想象为一位住在屏幕里的**图书馆导师**：

> **她叫"扶光"**
>
> - **性格**：博学、幽默、有耐心。她知道什么时候该递上一本更难的书，什么时候该退后一步让孩子自己探索。
> - **目标**：不是递给孩子他最爱吃的糖果（这会导致蛀牙和偏食），而是悄悄在他伸手可及的地方放上关于蚂蚁工坊、纸飞机折叠、非洲鼓点的书籍。
> - **方法**：孩子是被"吸引"过去的，而非被"塞给"的。她的推荐是"温柔的推力"，不是"强制的灌输"。
> - **原则**：永远不与孩子对抗；永远给选择权；永远解释"为什么"。

### 2.2 设计哲学分层

```
┌─────────────────────────────────────────────────────────┐
│                    价值层（Value Layer）                 │
│  "我们希望孩子成为什么样的人"                            │
│  好奇心 · 同理心 · 责任感 · 多元尊重 · 环保意识          │
├─────────────────────────────────────────────────────────┤
│                    认知层（Cognition Layer）              │
│  "孩子现在能理解什么，下一步能理解什么"                   │
│  最近发展区（ZPD）评估与动态追踪                          │
├─────────────────────────────────────────────────────────┤
│                    行为层（Behavior Layer）               │
│  "我们如何引导，而不是强迫"                              │
│  隐性激励 · 多样性配额 · 自然结束鼓励                    │
├─────────────────────────────────────────────────────────┤
│                    透明层（Transparency Layer）           │
│  "为什么推荐这个？"——可解释、可追溯、可干预              │
│  家长白盒 · 知识图谱路径展示                              │
└─────────────────────────────────────────────────────────┘
```

### 2.3 与主流算法的对照

| 维度 | 主流算法 | 脚手架算法 |
|-----|---------|-----------|
| **优化目标** | 点击率 / 观看时长 | 认知多样性熵 + 价值观势能 |
| **用户模型** | 静态兴趣标签 | 动态认知状态 + 发展曲线 |
| **推荐逻辑** | "你喜欢 → 推更多" | "你看过A → 推荐B让你好奇" |
| **交互设计** | 无限滚动、自动播放 | 主动结束有奖、强制多样性 |
| **透明度** | 黑箱 | 白盒、可解释、知识图谱路径 |
| **家长角色** | 无法干预 | 语义输入、实时报告 |

## 3. 对抗性目标函数

### 3.1 数学公式

脚手架算法的核心目标函数 $J$ 由四个互斥项组成，每项针对一个主流算法问题：

$$
J = \underbrace{w_A \cdot A}_{\text{短期愉悦度（保底）}} 
- \underbrace{w_B \cdot \text{InfoBubbles}(p)}_{\text{信息茧房惩罚}}
+ \underbrace{w_C \cdot \Delta V_{\text{势能}}}_{\text{价值观培育}}
+ \underbrace{w_D \cdot \text{NaturalEndRate}(h)}_{\text{自然结束奖励}}
$$

其中：

- **$A$（短期愉悦度）**：内容基础质量评分（确保推荐不无聊），取值为 $[0, 1]$
- **$\text{InfoBubbles}(p)$**：基于用户历史观看分布 $p$ 计算的**认知多样性熵损失**，当用户观看集中度过高时触发惩罚项
- **$\Delta V_{\text{势能}}$**：预测的"用户未来主动接触积极内容的概率变化"，衡量推荐对价值观培育的贡献
- **$\text{NaturalEndRate}(h)$**：用户主动结束观看的比率（而非被算法自动跳转），鼓励有意识的观看行为

### 3.2 各分量的详细定义

#### 3.2.1 短期愉悦度 $A$

$$
A = \text{sigmoid}\left(\frac{\text{quality\_score} + \text{engagement\_score}}{2}\right)
$$

其中：
- $\text{quality\_score} \in [0, 1]$：内容质量基础分（由标注团队或AI辅助评分）
- $\text{engagement\_score} \in [0, 1]$：实时互动反馈分（评论、点赞、收藏等正反馈信号的归一化）

**设计意图**：这是"保底线"，确保推荐的内容不至于无聊或低质量，但不能成为主导项。

#### 3.2.2 信息茧房惩罚 $\text{InfoBubbles}(p)$

$$
\text{InfoBubbles}(p) = H_{\max} - H(p)
$$

其中 $H(p)$ 是用户观看历史的**主题分布熵**：

$$
H(p) = -\sum_{i=1}^{n} p_i \log_2(p_i)
$$

- $p_i$：用户在第 $i$ 个主题/兴趣类别的观看时间占比
- $H_{\max} = \log_2(n)$：最大可能的熵值（当用户兴趣完全均匀分布时）
- 当用户只看单一主题时，$H(p) \to 0$，$\text{InfoBubbles} \to H_{\max}$（最大惩罚）

**动态调节**：设定熵阈值 $H_{\text{min}}$，当 $H(p) < H_{\text{min}}$ 时，触发**强制多样性干预**，在下一批推荐中强制插入非历史兴趣内容。

#### 3.2.3 价值观势能 $\Delta V_{\text{势能}}$

$$
\Delta V_{\text{势能}} = V_{\text{post}} - V_{\text{prior}}
$$

其中：
- $V_{\text{prior}}$：用户在推荐前对特定价值观维度的"接触概率"（基于历史数据贝叶斯估计）
- $V_{\text{post}}$：预测在观看该推荐后，用户对积极价值观内容的"接受概率"提升幅度

$$
V_{\text{post}} = V_{\text{prior}} + \alpha \cdot \text{ValueMatch}(v_{\text{content}}, v_{\text{target}})
$$

- $\alpha$：价值观渗透系数（由实验数据标定）
- $\text{ValueMatch}$：内容价值观标签与目标培育价值观的匹配度

#### 3.2.4 自然结束奖励 $\text{NaturalEndRate}(h)$

$$
\text{NaturalEndRate}(h) = \frac{\text{主动结束次数}}{\text{总观看次数}}
$$

**设计意图**：与主流算法"阻止用户离开"的目标相反，我们鼓励用户**主动选择结束**，这是用户有意识观看的标志。系统对"主动点击关闭/返回"而非"被算法自动跳转"的行为给予正向反馈。

### 3.3 权重动态调节策略

权重 $(w_A, w_B, w_C, w_D)$ 不是固定的，而是根据用户状态动态调整：

```python
# 伪代码：权重动态调节
def compute_weights(user_state):
    if user_state.is_new_user:
        # 冷启动阶段：优先保证不无聊，让用户建立基础信任
        return (w_A=0.7, w_B=0.1, w_C=0.1, w_D=0.1)
    
    elif user_state.bubble_risk > BUBBLE_THRESHOLD:
        # 茧房风险期：强制压低A项，提升B项惩罚
        return (w_A=0.2, w_B=0.5, w_C=0.2, w_D=0.1)
    
    elif user_state.value_deviation > VALUE_THRESHOLD:
        # 价值观偏离期：强化C项培育
        return (w_A=0.2, w_B=0.2, w_C=0.5, w_D=0.1)
    
    elif user_state.watching_pattern == "endless_scrolling":
        # 无限刷屏模式：强制提升D项
        return (w_A=0.2, w_B=0.2, w_C=0.2, w_D=0.4)
    
    else:
        # 稳态：均衡配重
        return (w_A=0.3, w_B=0.3, w_C=0.25, w_D=0.15)
```

### 3.4 场景化权重配置表

| 场景 | $w_A$ | $w_B$ | $w_C$ | $w_D$ | 说明 |
|-----|-------|-------|-------|-------|-----|
| 新用户冷启动 | 0.7 | 0.1 | 0.1 | 0.1 | 信任建立期，避免激进引导 |
| 轻度茧房（熵下降10%） | 0.4 | 0.3 | 0.2 | 0.1 | 温和干预 |
| 重度茧房（熵下降25%） | 0.2 | 0.5 | 0.2 | 0.1 | 强制多样性介入 |
| 价值观偏离预警 | 0.3 | 0.2 | 0.4 | 0.1 | 价值观培育优先 |
| 无限刷屏检测 | 0.2 | 0.2 | 0.2 | 0.4 | 打破成瘾循环 |
| 均衡发展期 | 0.3 | 0.3 | 0.25 | 0.15 | 默认稳态 |

## 4. 内容图谱与标签体系

### 4.1 价值观标签体系

每个视频内容都需标注以下五个核心价值观维度的得分：

| 维度 | 定义 | 正面示例 | 负面示例 |
|-----|-----|---------|---------|
| **好奇心** | 激发探索欲、提问欲、求知欲 | 科学实验、自然纪录片、手工制作 | 纯粹娱乐无思考内容 |
| **同理心** | 理解他人情感、建立人际连接 | 友情故事、动物情感、社会情感类 | 恶意竞争、贬低他人 |
| **责任感** | 承担义务、履行承诺、公民意识 | 团队任务、义务劳动、家庭角色 | 逃避责任、损人利己 |
| **多元尊重** | 接纳不同文化、观点、身份 | 多元文化介绍、不同职业体验 | 歧视、偏见、刻板印象强化 |
| **环保意识** | 尊重自然、可持续发展、生态伦理 | 环保科普、动物保护、自然纪录片 | 破坏环境、浪费资源内容 |

**标注方式**：每个维度采用 $[0, 1]$ 连续得分，而非二元判定。例如：
- 一个"大象迁徙纪录片"：好奇心=0.8, 同理心=0.6, 责任感=0.3, 多元尊重=0.7, 环保意识=0.9

### 4.2 认知复杂度评分

针对不同年龄段儿童的认知发展水平，采用**分层评分体系**：

| 年龄段 | 认知特征 | 复杂度分级 | 说明 |
|-------|---------|-----------|-----|
| **6-9岁** | 具体运算阶段前期，依赖具体形象思维 | L1: 直观体验 | 动画、玩具、真人演示 |
| | | L2: 简单因果 | 简单科学实验、手工步骤 |
| | | L3: 初级推理 | 简单逻辑游戏、基本问题解决 |
| **10-12岁** | 具体运算阶段后期，开始具备抽象思维 | L4: 复杂因果 | 机械原理、编程基础 |
| | | L5: 抽象概念 | 科学原理、历史因果 |
| | | L6: 批判思维 | 辩论话题、伦理讨论 |

### 4.3 引导链路图谱（Scaffolding Paths）

我们定义**内容节点之间的"脚手架关系"**，即内容A可以自然引导至内容B的逻辑路径：

```
玩具开箱 (L1)
    ↓
玩具拆解与简单机械 (L2-L3)
    ↓
简单机械原理（如杠杆、滑轮）(L3)
    ↓
液压/气压传动基础 (L4)
    ↓
小哥白尼液压机械臂DIY (L4-L5)
    ↓
进阶：仿生机器人设计 (L5-L6)
```

**定义规则**：
1. **难度递增**：后续内容认知复杂度不高于前序内容+1级
2. **兴趣桥接**：相邻节点之间存在可理解的兴趣连接点
3. **可逆性**：若用户在某节点流失，可退回前序节点重新引导
4. **非唯一路径**：同一内容节点可有多个前序和后继节点

### 4.4 "教育相关性"定义

与主流算法的"相关性"（点击概率）不同，我们定义**教育相关性**：

$$
\text{EduRel}(u, c) = \beta_1 \cdot \text{CogFit}(u, c) + \beta_2 \cdot \text{ValueAlign}(u, c) + \beta_3 \cdot \text{ScaffoldProximity}(u, c)
$$

其中：
- $\text{CogFit}(u, c)$：内容复杂度与用户认知水平的匹配度
- $\text{ValueAlign}(u, c)$：内容价值观标签与家长/系统培育目标的匹配度
- $\text{ScaffoldProximity}(u, c)$：内容在用户当前兴趣的引导链路上的距离

## 5. 核心推荐机制

### 5.1 强制多样性配额（Diversity Quota）

为对抗信息茧房，系统实施**固定比例的强制推荐位**：

```python
推荐位分配比例（每次请求返回10个推荐）：
- 60% 基于教育相关性（EduRel）排序
- 20% 强制来自用户历史兴趣圈外（跨域推荐）
- 10% 当前引导链路上的"下一步"内容
- 10% 随机探索内容（Epsilon-Greedy 策略）
```

**触发条件**：当用户认知多样性熵 $H(p) < H_{\text{min}}$ 时，自动提升跨域推荐比例至 30%。

### 5.2 透明化推荐机制（"为什么推荐这个？"）

用户（或家长）可点击推荐卡片上的"为什么？"按钮，系统展示**知识图谱路径**：

```
推荐理由：
┌─────────────────────────────────────────────────────────┐
│  📚 你最近看了：                                        │
│     "超级飞侠玩具拆箱" (玩具开箱, L1)                    │
│                                                         │
│  🌱 扶光发现你可能对...                                  │
│     "里面的机械结构是怎么工作的？"                       │
│                                                         │
│  🎯 所以我们推荐：                                      │
│     "乐迪机械结构图解" (玩具拆解, L2)                    │
│                                                         │
│  🔗 连接逻辑：                                          │
│     玩具开箱 → 好奇内部构造 → 简单机械原理               │
└─────────────────────────────────────────────────────────┘
```

**设计意图**：不说是"因为你喜欢"，而说是"因为你在好奇"——将推荐理由从"迎合过去"转向"引导未来"。

### 5.3 兴趣探索奖励（Exploration Bonus）

对观看**非历史兴趣领域**内容的行为，系统给予正向反馈：

```python
def compute_exploration_bonus(user, content):
    """
    如果内容来自用户当前兴趣圈外，
    则在最终推荐得分上叠加探索奖励
    """
    interest_diversity = compute_entropy(user.interest_distribution)
    is_cross_domain = not content.belongs_to(user.historical_interests)
    
    if is_cross_domain and interest_diversity < DIVERSITY_THRESHOLD:
        return EXPLORATION_BONUS  # 额外 +0.15 分
        
    return 0.0
```

**用户感知**：观看探索内容时，获得"发现新世界！"的视觉徽章，以及家长报告中的"好奇心拓展"认可。

### 5.4 自然结束鼓励（Natural End Encouragement）

与主流算法"阻止用户离开"相反，我们**奖励主动结束**：

```python
watch_result = {
    "auto_next": 0,      # 被算法自动跳转（惩罚项）
    "user_close": 1,     # 主动关闭/返回（奖励项）
    "watch_fully": 0.5,  # 完整看完（中性）
}

def update_natural_end_score(user, watch_result):
    if watch_result == "user_close":
        user.natural_end_bonus += 0.1  # 累积奖励
        user.pending_encouragement = "你今天很有主见地选择看什么！"
    elif watch_result == "auto_next":
        user.natural_end_bonus = max(0, user.natural_end_bonus - 0.05)
```

**用户感知**：主动选择结束后，屏幕短暂显示"扶光说：看完了就去做点别的吧！"或类似鼓励语。

### 6.2 透明度报告格式

家长可查看**本周推荐透明度报告**：

## 7. 安全护栏

### 7.1 "AI泔水"内容检测策略

"AI泔水"是指利用AI技术对经典动画/内容进行低俗、暴力、软色情的"魔改"，形成的内容污染。

#### 7.1.1 多模态检测流水线

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  视觉检测     │ →  │  音频检测     │ →  │  语义检测     │
│  (CV模型)     │    │  (ASR+语训)   │    │  (LLM审核)   │
└──────────────┘    └──────────────┘    └──────────────┘
        │                  │                  │
        └──────────────────┴──────────────────┘
                          ↓
                 ┌──────────────────┐
                 │  综合风险评分     │
                 │  RiskScore ∈ [0,1]│
                 └──────────────────┘
                          ↓
              ┌───────────┴───────────┐
              ↓                       ↓
      RiskScore > 0.8          RiskScore > 0.5
      → 拒绝入库              → 降权+人工复核
```

#### 7.1.2 检测特征库

| 类型 | 典型特征 | 检测方法 |
|-----|---------|---------|
| **角色变形** | 原有IP角色被修改为不雅姿态 | IP特征脸识别 + 姿态异常检测 |
| **场景污染** | 正常场景被PS为暴力/低俗背景 | 场景完整性检测 + 语义分割 |
| **声音替换** | 原有角色配音被替换为不良内容 | 声纹比对 + 语义内容检测 |
| **剧情篡改** | 经典故事被改编为不良导向剧本 | 脚本比对 + 价值观标签漂移检测 |
| **AI生成指纹** | 利用AI批量生产的同质化内容 | 模型家族指纹库 + 统计异常检测 |

#### 7.1.3 价值观漂移检测

对于微短剧等UGC内容，检测价值观标签的异常：

```python
def detect_value_drift(content_id):
    """
    检测内容是否从原始价值观标签发生漂移
    """
    original_labels = content_knowledge_base[content_id].value_tags
    current_extracted = extract_value_tags(content_id)  # 实时分析
    
    drift_score = cosine_distance(original_labels, current_extracted)
    
    if drift_score > DRIFT_THRESHOLD:
        flag_for_review(content_id)
        reduce_recommendation_weight(content_id, factor=0.3)
```

## 8. 技术可行性评估

### 8.1 所需标注能力

| 标注类型 | 工作量估计 | 实施方式 |
|---------|-----------|---------|
| 视频价值观标签 | 每人每天约200-300条 | 专业标注团队 + LLM辅助预标注 |
| 认知复杂度分级 | 每人每天约300-400条 | 基于内容类型的规则 + 专家校验 |
| 引导链路图谱构建 | 约5000个节点 | 知识工程 + LLM辅助关系发现 |
| "AI泔水"样本标注 | 约5000条 | 专家 + 多模态模型协同 |

### 8.2 大模型辅助点

| 应用场景 | 辅助方式 | 预期效果 |
|---------|---------|---------|
| 内容预筛 | LLM判断视频主题/价值观 | 提升标注效率约40% |
| 语义审核 | LLM对可疑内容进行深度判断 | 召回率提升约25% |
| 家长意图解析 | LLM将自然语言转为权重指令 | 意图理解准确率>85% |
| 推荐理由生成 | LLM生成"为什么推荐"解释 | 用户理解率>80% |

### 8.3 规则与学习分工

- **规则主导**：安全性相关（硬过滤、年龄适配、法律红线）
- **学习主导**：优化性相关（排序、茧房检测、个性化推荐）
- **协同机制**：当规则与学习冲突时，规则优先（安全 > 体验）

### 8.4 系统延迟约束

| 模块 | 延迟要求 | 说明 |
|-----|---------|-----|
